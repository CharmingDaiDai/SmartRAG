# SmartDoc 动画系统升级说明

## 📋 概述

本次升级全面丰富了 SmartDoc 的动画库，并将动画效果系统地应用到各个界面，提升了用户体验的流畅度和视觉吸引力。

- ✅ **SlideInUp** - 向上滑入动画

- ✨ **cardGridVariants** - 卡片网格动画变体（新增）
- ✨ **cardItemVariants** - 卡片项动画变体（新增）

## 🎬 动画设计原则

### 1. 性能优先
- 使用 CSS transform 和 opacity，避免触发重排
- 使用 GPU 加速的属性
- 合理的动画时长（0.2s-0.3s）

### 2. 流畅自然
- 使用弹簧物理引擎（spring）模拟真实运动
- cubic-bezier 缓动函数创造自然的加速度
- 错开动画时间（stagger）避免突兀

### 3. 视觉层次
- 重要元素先出现（标题 → 按钮 → 内容）
- 使用延迟创造节奏感
- 列表项依次出现增强层次

### 4. 交互反馈
- 悬停状态的即时反馈
- 点击时的按压感
- 加载状态的视觉指示

### 5. 克制优雅
- 避免过度动画
- 保持一致的动画风格
- 尊重用户的减少动画偏好设置

## 📊 性能优化

1. **组件级优化**
   - 使用 framer-motion 的优化引擎
   - 动画仅在必要时触发
   - 合理的动画时长避免卡顿

2. **CSS 优化**
   - 使用 will-change 提示浏览器
   - 避免动画复杂的属性（如 width/height）
   - 使用 transform 和 opacity 实现动画

3. **条件渲染**
   - 模态框动画仅在打开时渲染
   - 列表动画仅在数据加载完成后触发

## 📝 使用示例

```tsx
// 基础淡入
<FadeIn>
  <div>内容</div>
</FadeIn>

// 向上滑入（带延迟）
<SlideInUp transition={{ delay: 0.1 }}>
  <div>内容</div>
</SlideInUp>

// 卡片悬停效果
<HoverCard style={{ height: '100%' }}>
  <Card>内容</Card>
</HoverCard>

// 列表依次出现
<StaggerContainer>
  {items.map(item => (
    <StaggerItem key={item.id}>
      <Card>{item.name}</Card>
    </StaggerItem>
  ))}
</StaggerContainer>

// 模态框缩放动画
<Modal
  open={visible}
  modalRender={(modal) => <ScaleIn>{modal}</ScaleIn>}
>
  内容
</Modal>
```
