这份文档旨在为你提供一个清晰、可执行的 **SmartRAG 系统重构指南**。该方案专为**后端转全栈开发者**设计，核心原则是“少写 CSS，多用配置”，充分利用 Ant Design 生态的现成能力。

# SmartRAG 重构技术方案文档

## 1. 项目愿景

构建一个现代化的、企业级的 RAG（检索增强生成）管理与对话系统。

* **管理端**：高效管理知识库、文档上传与切片配置。
* **用户端**：流式对话体验，支持展示“检索意图”、“思维链”、“引用来源”等中间过程，并具备 Markdown 渲染能力。

## 2. 技术栈选型 (Tech Stack)

## 3. 目录结构规划

后端开发者喜欢清晰的分层，建议采用如下结构：

```text
src/
├── assets/             # 静态资源 (Logo, Lottie JSON)
├── components/         # 公共组件
├── hooks/              # 自定义 Hooks (useAuth, useStreamChat)
├── layouts/            # 布局文件
│   └── BasicLayout.tsx # 基于 ProLayout 的主壳子
├── pages/              # 页面入口
│   ├── Dashboard/      # 仪表盘
│   ├── KnowledgeBase/  # 知识库管理 (ProTable)
│   ├── Documents/      # 文档管理 (ProTable + Upload)
│   └── Chat/           # 智能问答 (AntD X)
├── services/           # API 接口层 (类似 Java 的 Controller 映射)
│   ├── kbService.ts
│   └── chatService.ts
├── store/              # 全局状态 (Zustand)
│   └── useAppStore.ts
├── types/              # TS 类型定义 (DTOs)
├── utils/              # 工具类 (request.ts, format.ts)
└── App.tsx             # 路由配置
```

## 4. 核心功能实现思路

### 4.1 全局布局 (Layout)

* **目标**：实现左侧导航、顶部面包屑、右侧用户信息。
* **实现**：使用 `ProLayout`。
* **配置**：
  ```tsx
  <ProLayout
    route={{
      routes: [
        { path: '/dashboard', name: '仪表盘', icon: <DashboardOutlined /> },
        { path: '/kb', name: '知识库', icon: <ReadOutlined /> },
        { path: '/chat', name: 'AI 对话', icon: <RobotOutlined /> },
      ]
    }}
    layout="mix"
    // ...
  >
    <Outlet /> {/* 子页面挂载点 */}
  </ProLayout>
  ```

### 4.2 管理模块 (知识库/文档)

* **目标**：实现增删改查表格、搜索、分页。
* **实现**：使用 `ProTable`。
* **亮点**：不需要手动处理 `loading` 和分页逻辑，只需将 `request` 属性绑定到一个返回 Promise 的函数即可。
* **操作列**：使用 `valueType: 'option'` 渲染操作按钮。

### 4.3 对话模块 (RAG Core) - 重难点

这是重构的深水区，需要前后端配合。

#### A. 后端接口协议 (SSE 建议)

Java 后端不应一次性返回 JSON，而应返回 `Content-Type: text/event-stream`。
事件流设计：

1. `event: thought` -> data: `{"stage": "searching", "msg": "正在检索 ES..."}`
2. `event: thought` -> data: `{"stage": "reranking", "msg": "检索完成，正在重排序..."}`
3. `event: message` -> data: `{"content": "根"}`
4. `event: message` -> data: `{"content": "据"}`
   ...

#### B. 前端渲染逻辑

使用 Ant Design X 的 `<Bubble.List>` 配合自定义的数据流处理。

```tsx
// 伪代码逻辑
const [messages, setMessages] = useState<Message[]>([]);

// 1. 收到 Thought 事件
// 找到最后一条 AI 消息，更新其 thoughts 字段
// <ThoughtChain items={currentMessage.thoughts} />

// 2. 收到 Message 事件
// 找到最后一条 AI 消息，追加 content 字段
// 界面自动渲染 Markdown
```

#### C. 动画增强

在 `Bubble.List` 的外层容器包裹 `AutoAnimate`，在组件挂载时使用 `Framer Motion` 做淡入。

## 5. 重构路线图 (Roadmap)

### 阶段一：脚手架与基础壳子 (1-2 天)

1. `npm create vite@latest smart-rag --template react-ts`。
2. 安装 `antd`, `@ant-design/pro-components`, `react-router-dom`。
3. 创建 `BasicLayout.tsx`，把侧边栏菜单配置好。
4. 配置 `react-router-dom`，点击菜单能跳转到空白页面。

### 阶段二：管理后台功能 (2-3 天)

1. **知识库管理**：使用 `ProTable` 实现列表展示。
2. **对接 Mock 数据**：先在前端写死数据，跑通交互。
3. **文档上传**：实现上传弹窗，使用 AntD 的 `<Upload.Dragger>`。

### 阶段三：AI 对话核心 (3-4 天)

1. 安装 `@ant-design/x`。
2. 实现静态的聊天界面布局（左边机器人，右边用户）。
3. **核心攻坚**：编写 SSE 请求工具类（可以使用 `fetch-event-source` 库）。
4. 实现 `ThoughtChain` 组件，展示“检索中 -> 排序中 -> 生成中”的状态变化。
5. 对接真实的 Java 后端接口。

### 阶段四：视觉优化 (1-2 天)

1. 引入 `Tailwind CSS` 调整间距和字体颜色。
2. 引入 `Framer Motion` 给页面切换加过渡。
3. 检查 Markdown 渲染（表格、代码块）是否美观。

## 6. 写给后端开发者的特别提示

1. **关于 CSS**：

   * 在 `ProComponents` 和 `AntD X` 中，你几乎不需要写 CSS。
   * 如果需要调整位置（比如“这个按钮离左边远一点”），**不要写 style 标签**，直接在 `className` 里写 `ml-4` (Tailwind 语法，意为 margin-left: 1rem)。这会让你的代码非常整洁。
2. **关于 State (状态)**：

   * 不要把所有变量都塞在 `App.tsx` 里。
   * 对话列表属于 `ChatPage` 的局部状态。
   * 用户信息属于 `Zustand` 的全局状态。
3. **关于调试**：

   * 安装 Chrome 插件 **React Developer Tools**。
   * 网络请求如果是 SSE 流，在 Chrome Network 面板里选 "Fetch/XHR"，点开请求看 "EventStream" 标签页，能看到实时的流数据，非常方便调试后端推流是否正常。
